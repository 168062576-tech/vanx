## [v4.2] - 2026-03-24

**回退重整 + 报告系统 + 演示模式 + 人口金字塔**

### 重构
- **Phase 1:** 回退到双服务架构（Web:5001 + API:5002），恢复稳定基础
- **Phase 2:** UI 拆分为 `web/index.html` + `css/main.css` + `js/app.js`，前后端分离
- **Phase 3:** 叙事引擎接入 API（中文名、生命故事、世界叙事、display API）

### 新增
- **报告系统（Task 1）：** 3 个报告 API（generate/smart/list）+ 报告中心页面 + 文件下载
  - 支持 HTML / JSON / Markdown 三种格式
  - 智能报告支持 executive / business / technical 三个级别
- **演示模式（Task 2）：** 90 秒电影式演示（1000 Agent × 12 月加速模拟）
  - 独立沙盒引擎，不影响主世界
  - 自动检测高光事件（首婚、首次犯罪、经济波动等）
- **人口金字塔（Task 3）：** `/api/v1/stats/pyramid` 端点 + ECharts 可视化

### 修复
- 修复 Jinja2 render_template_string 模板截断问题，改用直接 HTML 返回

## [v3.0] - 2026-03-23

**后端大重构 — 从 PoC 到工程化**

### 新增
- **SQLite 持久化层：** `storage.py`，32 项验证测试全部通过
- **LLM 混合决策引擎：** `decision_engine.py`，三级决策（L0 规则 / L1 本地 LLM / L2 线上 LLM），35 项测试通过
- **叙事引擎：** `narrative_engine.py`，Agent 生命故事生成，16 项测试通过
- **自动备份脚本：** 定时备份世界状态

### 修复
- **[致命] API 修复：** 移除对未定义 `world_manager` 的引用，改用 `DeepIntegrationEngine` 实例
- **[致命] O(N×M) 修复：** 公司遍历 bug 优化为 O(N+M)，修复 employees 类型错误
- **API Key 认证：** 新增 `X-API-Key` 中间件

### 重构
- 消除三套重复的 Agent 创建代码，统一到引擎，64 项集成测试全部通过
- 决策引擎集成到主引擎

### 文档
- 新增 `DESIGN.md` v3.0 系统设计文档

## [v2.0] - 2026-03-16

**初始版本 — 虚拟 Agent 世界 PoC**

### 新增
- **26+ 子系统：** 婚姻家庭、经济、企业、住房、医疗、社交网络、政府、深层微观（18 个子系统合并）
- **深度集成引擎：** `DeepIntegrationEngine`，统一 Agent 档案 + 事件系统
- **实验模板库：** 24 个实验模板（`experiment_templates_v2.py`）
- **Flask API：** RESTful 接口
- **Flask Web 可视化：** 基础仪表板
- **性能基准：** 10,000 Agent 每月模拟 ~0.4 秒（0.04ms/agent/月）

### 架构
- 纯内存存储（Dict[int, UnifiedAgent]）
- 双层子系统：P0 核心社会（5）+ P1 公共服务（3）+ P2/P3 深层微观（18）
- 统一事件流
